Here are some notes that I learned. Let's start from decorators.

What is a decorator in Python? A decorator is basically a function that wraps the function it decorates. The syntax of decorator is "@[decorator_name]" above the function it decorates.

1. The result of calling the decorated function is equivelent to decorator_func(decorated_function(arg1, arg2)), ..., etc.

2. The decorator could be placed on a class as well. If the class is decorated, it must have a __call__ function within the class.

3. A decorator could be declared within a class. You can define the function as a normal function instead of a class function or method. Then using function_name = staticmethod(function_name). Then you could use @[class_name].[decorator_name] to decorate outer functions.

4. The decorators are mostly used in: Logging, Authorization, Caching, Timing, Validation, Class Decorators, Frameworks and Libraries. Their ability to clearnly separate concerns makes them an essential tool for Python developers.

07/09/2024 What I did today.

I had an interview. The position is MLE and it is a coding interview which related to ML. The question is to implement K-Means algorithm.

So what is K-means? K-means is an algorithm that divide given n points into k clusters.

The process is like this: 1. Initialize k points as the centroid clusters. (There are different ways to do initialization) We can random pick k points to init as the centroids here.

2. Calculate the distances between each point and each centroid. Determine the centroid that each point belongs to by the distance. Make sure there is at least one point which belongs to each cluster.

3. Update the centroids by calculating the new centroid of each cluster.

4. Iterate from step 2 to step 4 until it hits the stopping criteria. The stopping point could be the cluster each point belongs to doesn't change from previous one.
